Welcome to the next version of the CollaboRate work breakdown wizard

This tool will guide you to break down your assigned task into a maximum of two subtasks. You must not solve the assigned task completely. All you must do is create subtasks such that these subtasks when taken together, exactly cover the scope and information present in the assigned task (nothing less or more, except that the subtasks might elaborate details that were implied by the parent task.

Instructions for Crowdsourcing Vendors
Once the following are identified:
  • Sub-part 1 of the problem statement
  • Sub-part 2 of the problem statement
  • Logic that connects the 2 sub-parts
...you need to create 3 nodes for each of the above in the db via calling the appropriate js functions from the client side. For each of these three nodes, the problem statement that was broken down using this tool is the parent node.

Application Interactions

The problem statement describes inputs provided to the application that is being developed for this task. It also describes how the application responds to these inputs.
